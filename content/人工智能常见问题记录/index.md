## 问题 1：调用业务接口（`/HFB` 开头）失败

### 页面现象

### 可能原因

1. 对应的公积金业务后台 Java 程序未同步更新。
2. 公积金后台存在异常，或存储脚本初始化失败。

### 排查与处理

1. 通过控制台查看对应接口返回结果。
2. 如果返回 `404`，通常表示公积金后台尚未同步更新，需要先完成后台程序更新。
3. 检查公积金后台日志，确认是否存在对应异常。
4. 如果涉及存储或初始化报错，参考人工智能服务部署手册，确认初始化脚本是否执行成功。

## 问题 2：现场调用公司平台业务环境 Dify 网络异常

### 页面现象

![](/人工智能常见问题记录/assets/常见问题原因及处理方案/file-20260410095858427.png)

![](/人工智能常见问题记录/assets/常见问题原因及处理方案/file-20260410095915952.png)

### 可能原因

1. 现场网络未打通，无法访问 Dify 服务。
2. Dify 配置中的 `api_url` 或 `api_key` 填写错误。

### 处理方案

#### 方式一：逐笔更新

进入以下菜单后，编辑对应 AI 工作流配置并更新 `api_url` 与 `api_key`：

`运维服务商 -> 人工智能配置 -> 页面设置 -> AI 工作流 -> 编辑`

#### 方式二：批量更新

进入对应 Docker 容器后，连接 SQLite 数据库并检查配置：

```bash
docker exec -it damoxing-app sh
sqlite3 database.sqlite
.head on
select * from sys_dify_config limit 1;
```

返回结果中重点关注以下字段：

- `page_key`
- `workflow_name`
- `api_url`
- `api_key`
- `enabled`

确认目标记录后，执行更新语句：

```sql
UPDATE sys_dify_config
SET api_url = 'https://<your-dify-domain>/v1',
    api_key = 'app-<your-api-key>'
WHERE page_key = '<page_key>';
```

### 注意事项

1. 发布到公开站点前，不要保留真实 `api_key`、内网地址或生产环境配置。
2. 执行数据库更新前，先确认当前容器和数据库环境无误。
3. 如果需要批量修改，建议先查询目标记录，再执行更新，避免误改其他页面配置。
